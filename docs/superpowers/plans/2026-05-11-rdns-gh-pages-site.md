# rDNS GitHub Pages Marketing Site — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a 7-page Jekyll marketing site for rDNS at `docs/`, deployed via GitHub Pages, optimized for SEO and broad-audience appeal, with Lighthouse ≥95 across all categories.

**Architecture:** Native Jekyll site (no GH Actions) using `_layouts` / `_includes` for DRY structure, system-font stack for zero-RTT typography, vanilla JS <5KB, CSS custom properties for dark/light theming. Content authored as markdown front-matter pages; static facts (version, comparison matrix) live in `_data/` YAML.

**Tech Stack:** Jekyll 3.10 (github-pages gem), SCSS, vanilla JS (ES2020), Liquid templating, JSON-LD structured data.

**Spec:** `docs/superpowers/specs/2026-05-11-rdns-gh-pages-design.md`

---

## Decisions locked in (resolves spec's open questions)

- **Feature icons** — hand-rolled stroke SVGs (Lucide-style), inlined via `_includes/icons/*.svg` to avoid extra HTTP requests.
- **OG image** — one shared 1200×630 card (`assets/img/og-default.png`) generated by running `chromium --headless --screenshot` against a one-shot HTML template (`assets/og-template.html`). Per-page variants deferred to phase 2.
- **Version display** — `site.version` set in `_config.yml`, updated alongside `Cargo.toml` version bumps per project convention.

```javascript
(function () {
  'use strict';

  // Theme toggle
  var root = document.documentElement;
  var themeBtn = document.querySelector('.theme-toggle');
  if (themeBtn) {
    themeBtn.addEventListener('click', function () {
      var current = root.getAttribute('data-theme');
      var next = current === 'light' ? 'dark' : 'light';
      root.setAttribute('data-theme', next);
      try { localStorage.setItem('theme', next); } catch (e) {}
    });
  }

  // Mobile nav toggle
  var nav = document.querySelector('.site-nav');
  var navToggle = document.querySelector('.nav-toggle');
  if (nav && navToggle) {
    navToggle.addEventListener('click', function () {
      var open = nav.classList.toggle('is-open');
      navToggle.setAttribute('aria-expanded', open ? 'true' : 'false');
    });
  }

  // Copy-to-clipboard for code blocks
  document.querySelectorAll('pre > code').forEach(function (code) {
    var pre = code.parentElement;
    if (pre.querySelector('.copy-btn')) return;
    var btn = document.createElement('button');
    btn.className = 'copy-btn';
    btn.type = 'button';
    btn.setAttribute('aria-label', 'Copy code');
    btn.textContent = 'Copy';
    btn.addEventListener('click', function () {
      navigator.clipboard.writeText(code.textContent).then(function () {
        btn.textContent = 'Copied';
        setTimeout(function () { btn.textContent = 'Copy'; }, 1500);
      });
    });
    pre.appendChild(btn);
  });

  // Tabbed install snippets
  document.querySelectorAll('.tabs').forEach(function (tabs) {
    var triggers = tabs.querySelectorAll('[role="tab"]');
    var panels = tabs.querySelectorAll('[role="tabpanel"]');
    triggers.forEach(function (trigger) {
      trigger.addEventListener('click', function () {
        var id = trigger.getAttribute('aria-controls');
        triggers.forEach(function (t) { t.setAttribute('aria-selected', 'false'); });
        panels.forEach(function (p) { p.hidden = (p.id !== id); });
        trigger.setAttribute('aria-selected', 'true');
      });
    });
  });

  // Animate benchmark bars on scroll-into-view
  var bars = document.querySelectorAll('.bar[data-pct]');
  if (bars.length && 'IntersectionObserver' in window) {
    var io = new IntersectionObserver(function (entries) {
      entries.forEach(function (entry) {
        if (entry.isIntersecting) {
          var el = entry.target;
          el.style.width = el.getAttribute('data-pct') + '%';
          io.unobserve(el);
        }
      });
    }, { threshold: 0.2 });
    bars.forEach(function (b) { io.observe(b); });
  } else {
    bars.forEach(function (b) { b.style.width = b.getAttribute('data-pct') + '%'; });
  }
})();
```

    <h3>v1 → v2: Concurrency (+216%)</h3>
    <p>The original UDP listener processed queries sequentially. Each query blocked the socket while waiting for upstream resolution.</p>
    <p><strong>Fix:</strong> Spawn a Tokio task per incoming query. Forwarder connection pool multiplexes queries over one connected UDP socket with oneshot-channel response dispatch.</p>

    <h3>v2 → v3: Reduce allocations (−9%, better scaling)</h3>
    <p>Spawning a task per query added clone overhead and full <code>Message::decode → encode</code> round trips.</p>
    <p><strong>Fix:</strong> Sync fast-path for cache hits, authoritative answers, and RPZ blocks — inline in the recv loop. Wire-format fast parser. Direct wire encode with TTL adjustment.</p>

    <h3>v3 → v4: Faster cache (+264%)</h3>
    <p>DashMap's coarse sharding and <code>get_mut</code> write-lock-on-hit became the bottleneck.</p>
    <p><strong>Fix:</strong> Custom 256-shard cache on <code>parking_lot::RwLock</code>. Cache hits take read locks only. LTO fat, single codegen unit, target-cpu native in the release profile.</p>

    <h3>v4 → v5: Eliminate socket contention (+41%)</h3>
    <p>Multiple workers sharing one socket caused kernel-level race on the receive buffer.</p>
    <p><strong>Fix:</strong> <code>SO_REUSEPORT</code> — separate socket per worker on the same port. Kernel distributes packets by flow hash. <code>SO_RCVBUF</code> increased to 4 MB.</p>

- Programmatic version sync from Cargo.toml — `site.version` is manually updated on releases.
- Custom domain — staying on default GitHub Pages URL.
- CI test workflow — Jekyll builds in GH Pages; a broken build is loud.
